## ✨ Идея продукта

Большинство шахматных платформ уже умеют давать пользователю доску и рейтинговые партии. **Chess Prime** фокусируется на том, чего часто не хватает игрокам:

- прозрачности вокруг жалоб и честной игры;
- персонального обучения после каждой партии;
- комфортного онбординга для новичков;
- современных клубов, лиг и командных сценариев;
- быстрого и чистого интерфейса без перегруза.

Полная продуктовая стратегия лежит в [`docs/online-chess-product-plan.md`](docs/online-chess-product-plan.md).

## 🖥️ Быстрый старт

### Требования

- Любой современный браузер для открытия `index.html` напрямую
- Node.js/npm нужны только для npm-скриптов `start`, `build` и `preview`

### Установка

```bash
npm install
```

### Запуск dev-сервера

```bash
npm start
```

После запуска откройте адрес локального сервера:

```text
http://localhost:5173
```

### Production build

```bash
npm run build
```

### Preview production build

```bash
npm run preview
```

## 🧠 Архитектура MVP

Сейчас приложение полностью клиентское и может открываться напрямую через `index.html` без React-сборки:

1. `src/main.js` хранит состояние доски, выбранную фигуру, сторону хода, историю ходов, online/offline режим, статус бота и статус жалобы.
2. `generateMoves` фильтрует ходы так, чтобы король не оставался под шахом, а `updateGameEndState` определяет шах, мат и пат.
3. `makeMove` применяет ход и делает автоматическое превращение пешки в ферзя на последней горизонтали.
4. `evaluate` считает простой материальный баланс для панели тренера.
5. UI-компоненты показывают доску, online/offline-панель, историю, coach-панель, переключатель звука, roadmap и карточки продуктовых преимуществ.

## 🛣️ Roadmap

### 1. Игровое ядро

- [x] Проверка шаха, мата и пата
- [x] Рокировка, взятие на проходе, выбор фигуры при превращении
- [x] PGN export и FEN import/export
- [x] Таймеры и завершение партии по времени

### 2. Онлайн-режим

- [ ] WebSocket-сервер для real-time партий
- [ ] Server-authoritative clock
- [ ] Reconnect без потери партии
- [ ] Очереди bullet/blitz/rapid

### 3. Fair Play

- [ ] Очередь жалоб и модераторский кабинет
- [ ] Статусы жалоб для пользователя
- [ ] Компенсация рейтинга при подтвержденном нарушении
- [ ] Verified ranked queue для серьезных партий

### 4. Обучение

- [ ] Stockfish-анализ завершенной партии
- [ ] Топ-3 критических момента партии
- [ ] Персональные пазлы из ошибок
- [ ] Недельный тренировочный план

### 5. Комьюнити

- [ ] Клубы и командные матчи
- [ ] Арены и Swiss-турниры
- [ ] Study rooms для совместного разбора
- [ ] Инструменты тренеров и капитанов

## 🎯 Метрики, на которые целимся

- median matchmaking time меньше 5 секунд для популярных контролей;
- game completion rate выше 92%;
- рост D1/D7 retention за счет coach-подсказок и мягкого онбординга;
- понятный SLA по жалобам и прозрачная компенсация рейтинга;
- высокая доля игроков, которые после партии запускают рекомендованную тренировку.

## 🤝 Как вносить изменения

1. Создайте ветку под задачу.
2. Внесите изменения в `src/` или `docs/`.
3. Проверьте сборку:

```bash
npm run build
```

4. Опишите в PR, какую часть продукта улучшили: gameplay, fair play, обучение, комьюнити, инфраструктуру или документацию.

## ⚠️ Ограничения текущего прототипа

- Нет backend и настоящего онлайн-матчмейкинга: онлайн-поиск пока demo-flow, зато офлайн-бот работает локально.
- Coach-панель пока использует простую материальную оценку, а не шахматный движок.
- Fair Play workflow пока демонстрационный и не подключен к модерации.

## 📌 Видение

Мы строим не просто «еще одну доску», а шахматную платформу, где игрок понимает:

- почему он проиграл;
- что ему тренировать дальше;
- что происходит с жалобами;
- как играть в сильном и здоровом комьюнити.

**Chess Prime должен ощущаться как личный шахматный клуб, тренер и честная турнирная площадка в одном продукте.**
